# Docs Wiki 维护规则（KISS）

## 1) 写入位置（SSOT）
- 现行文档写在对应主题目录（如 `docs/sse/`、`docs/api-contracts/`、`docs/runbooks/`）。
- 阶段交付/一次性材料/历史方案统一归档到 `docs/archive/`。

## 2) 命名
- 新增文档优先使用 `kebab-case.md`（英文/数字），避免路径兼容性问题。
- 若必须中文命名，保持简短并只在目录层级出现一次。

## 3) 链接
- 站内引用统一使用相对路径（`./` / `../`），避免硬编码盘符路径。
- “历史文档”允许保留当时的命令/路径，但需要在文档顶部补一行“现状路径”提示。

## 4) 归档（Archive）
- 归档动作只做：`git mv` + 修正文档导航与关键引用。
- 不改写历史结论；如现状已变化，在顶部追加“现状说明”小节。
